Resumo rápido do tema

Os pilares que você precisa dominar primeiro.

Objeto

Estrutura que agrupa dados em pares chave: valor.

Propriedade

É cada dado guardado dentro do objeto.

Método

É uma função armazenada dentro do objeto.

this

Em métodos tradicionais, costuma apontar para o próprio objeto.

1) O que é um objeto

O ponto de partida do tema.

Em JavaScript, um objeto é uma forma de guardar várias informações relacionadas dentro de uma única estrutura.

Em vez de criar várias variáveis separadas, podemos reunir tudo dentro de um único objeto. 

const pessoa = {
    nome: "João",
    idade: 48,
    cidade: "Ribeirão Preto"
};
  • pessoa é o objeto
  • nome, idade e cidade são as chaves
  • Os dados à direita são os valores

2) Propriedades do objeto

São as características armazenadas dentro dele.

const pessoa = {
    nome: "João",
    apelido: "Ribeiro",
    idade: 48,
    genero: "masculino"
};
  • nome é uma propriedade
  • apelido é uma propriedade
  • idade é uma propriedade
  • genero é uma propriedade

3) Métodos

Funções que vivem dentro do objeto.

const pessoa = {
    nome: "João",
    apelido: "Ribeiro",

    apresentarNome: function () {
        return this.nome + " " + this.apelido;
    }
};

Para executar o método: 

pessoa.apresentarNome();

4) Entendendo o this

Um dos pontos mais importantes do assunto.

apresentarNome: function () {
    return this.nome + " " + this.apelido;
}
Atenção: no começo, use função tradicional em métodos de objeto. Arrow function com this costuma confundir.

5) Acessando propriedades

Ponto e colchetes.

Notação de ponto

pessoa.nome
pessoa.idade

Notação de colchetes

pessoa["nome"]
pessoa["idade"]

Colchetes são úteis quando o nome da propriedade vem de uma variável. 

const propriedade = "nome";
console.log(pessoa[propriedade]);

6) Arrays e objetos dentro de objetos

Estruturas aninhadas.

const pessoa = {
    nome: "João",
    hobbies: ["Programação", "aqui aparece na lousa é isso?" "Cinema", "Música"],
    endereco: {
        cidade: "Ribeirão Preto",
        estado: "SP"
    }
};
pessoa.hobbies[0]
pessoa.endereco.cidade
pessoa.endereco.estado

7) Percorrendo com for...in

Como listar as chaves do objeto.

for (let propriedade in pessoa) {
    console.log(propriedade);
    console.log(pessoa[propriedade]);
}

8) Object.keys(), Object.values() e Object.entries()

Ferramentas úteis para inspecionar objetos.

Object.keys(objeto)

Retorna um array com as chaves.

Object.values(objeto)

Retorna um array com os valores.

Object.entries(objeto)

Retorna pares [chave, valor].

9) Adicionando propriedades

Objetos são flexíveis.

pessoa.email = "joao@gmail.com";
pessoa["telefone"] = "(16) 99999-9999";

10) Alterando valores

O valor antigo é substituído pelo novo.

pessoa.idade = 49;
pessoa.nome = "João Pedro";

11) Removendo propriedades com delete

Exclusão direta da chave.

delete pessoa.genero;

12) Objetos são copiados por referência

Um dos pontos que mais causam erro.

const transporte = {
    tipo: "automóvel",
    ano: 2032,
    marca: "Audi"
};

const novoTransporte = transporte;
novoTransporte.marca = "Mercedes";
Resumo: os dois nomes apontam para o mesmo objeto.

13) Copiando com Object.assign()

Cópia rasa do primeiro nível.

const copia = Object.assign({}, objetoOriginal);

14) Copiando com spread operator

Forma moderna e muito usada.

const copia = { ...objetoOriginal };

15) O que é cópia rasa

O primeiro nível copia; os internos ainda podem ficar ligados.

const usuarioOriginal = {
    nome: "Carlos",
    endereco: {
        cidade: "Ribeirão Preto",
        estado: "SP"
    }
};

const usuarioCopia = { ...usuarioOriginal };

usuarioCopia.endereco.cidade = "Sertãozinho";
Atenção: o objeto interno endereco continua compartilhado.

Resumo final para fixar

Leitura rápida para revisão.

  • Objeto agrupa dados em chave: valor
  • Propriedade é o dado guardado
  • Método é uma função dentro do objeto
  • this costuma apontar para o próprio objeto
  • Ponto e colchetes acessam propriedades
  • for...in percorre chaves
  • Object.keys, values e entries ajudam na inspeção
  • delete remove propriedade
  • Objetos são atribuídos por referência
  • Object.assign e spread fazem cópia rasa
